# WirelessID - 无线充电系统通讯板

## 项目目标

本工程是**无线充电系统的外置通讯板**，用于无线识别和信息交互。

### 系统架构

**一发四收**无线充电系统：
- 1 个发射端（发送端功率板 + 基站端通讯板）
- 4 个接收端（接收端功率板 + 设备端通讯板）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        一发四收无线充电系统                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │                        发射端                                    │   │
│   │   ┌──────────────┐      CAN       ┌──────────────┐             │   │
│   │   │  发送端功率板  │ ←────────────→ │  基站端通讯板  │             │   │
│   │   │  (充电发射)    │               │  (ID: 0xA0)  │             │   │
│   │   └──────────────┘               └──────┬───────┘             │   │
│   └─────────────────────────────────────────┼─────────────────────┘   │
│                                             │                         │
│                              ┌──────────────┼──────────────┐          │
│                              │              │              │          │
│                              ↓              ↓              ↓          │
│   ┌────────────────────┐ ┌────────────────────┐ ┌────────────────────┐│
│   │     接收端 #1       │ │     接收端 #2       │ │   接收端 #3/#4     ││
│   │ ┌────────────┐     │ │ ┌────────────┐     │ │ ┌────────────┐     ││
│   │ │接收端功率板 │     │ │ │接收端功率板 │     │ │ │接收端功率板 │     ││
│   │ │(充电接收)   │     │ │ │(充电接收)   │     │ │ │(充电接收)   │     ││
│   │ └─────┬──────┘     │ │ └─────┬──────┘     │ │ └─────┬──────┘     ││
│   │       │CAN         │ │       │CAN         │ │       │CAN         ││

│   通讯方式:                                                             │
│   1. 红外透传 CAN 数据 (20Hz, 基站请求-设备回复)                         │

│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 系统组成

| 组件 | 数量 | 角色 | 说明 |
|------|------|------|------|
| 发送端功率板 | 1 | 充电发射 | 通过 CAN 连接基站端 |
| 基站端通讯板 | 1 | 主控/网关 | 协调 4 个设备端通讯 |
| 接收端功率板 | 4 | 充电接收 | 通过 CAN 连接设备端 |
| 设备端通讯板 | 4 | 终端/中继 | 连接接收端，与基站通讯 |

### 关键需求

| 需求 | 说明 |
|------|------|
| 基站数量 | 1 个（ID: 0xA0） |
| 设备数量 | 最多 4 个（ID: 0xD0-0xD3） |
| 通讯模式 | **单向**：设备端只发送，基站端只接收 |
| 设备识别 | 设备 ID 在配置中指定 |
| CAN 速率 | 1Mbps |
| 红外速率 | 4800 baud |
| 发送频率 | 20Hz（50ms 周期） |
| 数据过期 | 100ms 无新数据则停止发送 |

### 通讯流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        单向红外通讯流程                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  设备端                              基站端                              │
│    │                                   │                                │
│    │  持续接收功率板CAN报文               │                                │
│    │  缓冲区: [CAN数据 + 时间戳]         │                                │
│    │                                   │                                │
│    │  每 50ms 检查缓冲区                 │                                │
│    │  如果有新数据（<100ms）：            │                                │
│    │── IR RSP [CAN数据] ──────────────→│                                │
│    │                                   │  解析CAN数据                    │
│    │                                   │  转发到发送端功率板              │
│    │                                   │                                │
│    │  如果无新数据：                     │                                │
│    │  (跳过，不发送)                     │                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 工程概述

基于 **ESP32-C3 + ESP-IDF v5.4** 的嵌入式工程，支持 CAN / 红外 / ESP-NOW / I2C 屏幕 / WS2812B 多模块协同。

## 当前状态

### 通信协议层（已完成）

| 模块 | 状态 | 说明 |
|------|------|------|
| TWAI (CAN) | ✅ | 1Mbps，bus-off 自动恢复，WebUI 实时监控 |
| 红外串口 | ✅ | UART1 + 38kHz PWM，4800 波特率稳定 |
| 红外主从协议 | ✅ | CMD_REQ/RSP 双向，~65% 成功率 |
| ESP-NOW 组网 | ✅ | 设备发现/心跳/CMD_REQ-RSP，1Hz 稳定 |
| CAN-IR 桥接 | ✅ | TX→IR→RX→TWAI 链路验证通过 |

### 硬件驱动层（已完成）

| 模块 | 状态 | 说明 |
|------|------|------|
| I2C / OLED | ✅ | SSD1306 128×64，I2C DMA，旋转支持 |
| WS2812B | ✅ | RMT 驱动 + 效果引擎（SOLID/BREATH/RAINBOW） |
| UART0 TX | ✅ | 发送稳定，事件驱动接收 |
| WebUI 仪表盘 | ✅ | Wi-Fi AP + HTTP，实时状态监控 |
| KEY 按键 | ⚠️ | 仅 GPIO 初始化骨架 |

### 应用层（已完成）

| 模块 | 状态 | 说明 |
|------|------|------|
| 命令调度框架 | ✅ | 统一命令注册/路由/响应，支持多链路 |
| 系统命令 | ✅ | 查询状态/版本/统计、重启、配置 |
| 设备控制 | ✅ | WS2812 灯光控制（颜色/效果/亮度） |
| 显示控制 | ✅ | OLED 清屏/文本显示/刷新 |
| 输入设备 | ✅ | 按键查询/事件（骨架） |
| 红外单向通讯 | ✅ | 设备端发送 CAN 数据，基站端接收转发 |

### 业务层（进行中）

| 模块 | 状态 | 说明 |
|------|------|------|
| 红外单向通讯 | ✅ | 设备→基站单向 CAN 透传，3m 稳定 |
| 缓冲区优化 | ✅ | 时间戳机制，100ms 数据过期检查 |
| OLED 状态显示 | ✅ | 显示 CAN ID 和数据 |

## 命令框架快速参考

### 启用测试

在 `project_config.h` 中设置：
```c
#define WIRELESSID_CMD_TEST_ENABLE 1
```

### UART 命令格式

通过串口终端发送十六进制命令：

| 命令 | 字节序列 | 说明 |
|------|----------|------|
| 查询状态 | `01` | 返回设备ID、运行时间 |
| 查询版本 | `03` | 返回版本号、设备名 |
| 查询统计 | `04` | 返回命令统计信息 |
| 重启 | `02` | 重启设备 |
| LED 红色 | `10 FF 00 00` | R=255, G=0, B=0 |
| LED 绿色 | `10 00 FF 00` | R=0, G=255, B=0 |
| 呼吸灯效果 | `11 02 00 FF 00 80` | effect=BREATH, R=0, G=255, B=0 |
| OLED 清屏 | `20` | 清除显示 |
| OLED 显示文本 | `21 00 00 48 65 6C 6C 6F` | row=0, col=0, "Hello" |

### 命令类型码

| 范围 | 用途 |
|------|------|
| 0x01-0x0F | 系统命令 |
| 0x10-0x1F | 设备控制（LED） |
| 0x20-0x2F | 显示控制（OLED） |
| 0x30-0x3F | 输入设备（按键） |
| 0x40-0x4F | 桥接命令 |
| 0xF0-0xFF | 用户自定义 |

详细文档见 `doc/cmd_design.md`。
